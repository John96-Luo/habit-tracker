# 每日三件事 · 产品需求文档 (PRD)

> 版本：v1.0 | 日期：2026-07-20 | 状态：MVP 已交付

---

## 1. 产品概述

**每日三件事** 是一个极简的习惯追踪 Progressive Web App，帮助用户每天聚焦最重要的三项任务，通过轻量复盘建立持续行动力。

### 1.1 核心价值主张

> "少即是多" —— 每天只做三件最重要的事，用最小的行动成本撬动最大的成就感。

- **输入极简**：三个输入框，写完即走，无需点击保存
- **反馈即时**：勾选即记录，所有操作毫秒级响应
- **隐私至上**：数据仅存本地浏览器，无需登录、无云端同步
- **回溯有力**：周/月统计 + 日历热力图，直观看到积累的力量

### 1.2 目标用户

| 用户画像 | 痛点 |
|----------|------|
| 被繁重待办列表压垮的人 | 任务过多导致决策瘫痪 |
| 希望养成每日复盘习惯的人 | 复盘工具过于复杂 |
| 追求极简行动哲学的个人用户 | 需要轻量、零摩擦的工具 |

## 2. 功能需求

### 2.1 今日三件事（首页）

**输入**
- 三个输入框，每个占位文字依次为「第1件最重要的事是…」「第2件最重要的事是…」「第3件最重要的事是…」
- 用户输入时，300ms 防抖后自动保存至 localStorage（无需提交按钮）
- 输入框失焦时立即保存

**标记完成**
- 每项右侧显示圆形勾选框，点击切换完成/未完成
- 已完成事项文字显示删除线，整行降低透明度

**优先级调整**
- 拖拽手柄（⋮⋮）支持上下拖拽排序
- 使用 HTML5 Drag & Drop API 原生实现
- 拖拽完成后立即持久化新的顺序

**周进度预览**
- 页面底部展示本周完成进度条：「本周 9/21」
- 进度条动态显示完成比例（绿色渐变填充）

**跨午夜处理**
- 打开今日页时自动检测日期，进入新一天后自动创建新的空白记录

### 2.2 历史记录页

- 按日期倒序显示所有记录（最新在上）
- 每条记录以卡片形式展示：
  - 日期 + 完成状态（如「✅ 2/3」）
  - 三件事的完整内容及勾选状态
  - 删除按钮（点击后弹窗确认）
- 无数据时显示空状态引导文案

### 2.3 统计页

**本周进度**
- 大号数字展示：`已完成件数 / 21`
- 进度条可视化 + 完成率百分比

**月度热力图**
- 7 列 × N 行日历网格（周一起始）
- 4 级颜色深度表示当日完成情况：

  | 完成数 | 颜色 | CSS 类 |
  |--------|------|--------|
  | 0/3 | 浅灰 `#f0f0f0` | `cell-empty` |
  | 1/3 | 浅绿 `#c8e6c9` | `cell-low` |
  | 2/3 | 中绿 `#66bb6a` | `cell-mid` |
  | 3/3 | 深绿 `#2e7d32` | `cell-high` |

- 点击格子显示日期和完成详情（title 悬浮提示）
- 当天格子有深色边框突出显示
- 支持前后月份切换
- 月度汇总：本月完成件数 + 日均完成率

### 2.4 数据管理

- **导入导出**：导出为格式化的 JSON 文件，支持重新导入恢复
- **数据版本**：JSON 结构包含 `version` 字段，预留未来数据迁移能力

## 3. 非功能需求

| 指标 | 目标 | 现状 |
|------|------|------|
| 首屏加载 (FCP) | < 1 秒 | ~798ms（开发环境） |
| 交互响应 | < 50ms | 即时（零网络请求） |
| 移动端适配 | 375px 起 | 响应式设计，最大宽度 480px |
| 打包体积 (gzip) | < 15KB | 预计 < 15KB（UnoCSS 按需生成） |

### 3.1 用户体验原则

- 所有交互即时响应，无网络等待（纯本地存储）
- 移动端优先，单手可操作
- 安全区域适配（`env(safe-area-inset-bottom)`）
- 禁用页面缩放（`user-scalable=no`），防止输入时意外缩放

```
│  今天最重要的三件事            │  ← Header

├─────────────────────────────┤
│  ⋮⋮ [第1件最重要的事是…]    ○ │
│  ⋮⋮ [第2件最重要的事是…]    ○ │  ← 三个 TodoItem（输入+勾选+拖拽）
│  ⋮⋮ [第3件最重要的事是…]    ○ │
├─────────────────────────────┤

```

## 6. 设计规范

| 项目 | 值 |
|------|------|

## 7. 术语表

| 术语 | 定义 |
|------|------|
| TodoItem | 单件事项，包含 id、内容、完成状态、排序位 |
| DayRecord | 某天的完整记录，固定包含 3 个 TodoItem |
| AppData | 顶层数据结构，包含版本号和所有 DayRecord |
| WeeklyStats | 周统计（纯计算，不持久化） |
| MonthlyStats | 月统计（纯计算，不持久化） |
